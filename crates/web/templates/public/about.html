Liquid Metal

A serverless platform for developers who ship compiled code. Deploy a binary, get a URL, pay per request.

{# Problem #}

// problem

Serverless forgot about compiled languages

Lambda wants a runtime. Vercel wants JavaScript. Cloudflare Workers want V8. If you write Go, Rust, or Zig, you already have a binary — you shouldn't need to wrap it in a container or an SDK just to deploy it.

{# Approach #}

// approach

Two engines, one experience

Metal runs your binary in a Firecracker microVM. On deploy, we boot it, verify it works, snapshot the VM state, and shut it down. On the first request, we restore the snapshot in <15ms. Real Linux. Real isolation. Scales to zero.

Liquid compiles your code to WebAssembly and runs it in Wasmtime. Sub-millisecond cold starts. No VM, no kernel, no filesystem. For APIs, webhooks, and data transforms that don't need syscalls.

{# How it works #}

// how it works

Deploy

flux deploy builds your binary, uploads it, and prepares it for execution. No Dockerfiles. No YAML. One command.

Scale to zero

After 5 minutes of inactivity, your service sleeps. No VM running. No memory reserved. You pay nothing until the next request.

Wake on request

When a request arrives, your service wakes from snapshot (<15ms Metal) or cold-starts (<1ms Liquid). The user gets a response, not a loading screen.

{# Stack #}

// under the hood

Firecracker + Snapshots

Metal services run in KVM-isolated microVMs. On deploy, we snapshot the fully booted app. On request, we restore from snapshot — no cold boot.

eBPF Tenant Isolation

Per-VM network isolation via TC egress classifiers. Cross-tenant traffic is blocked at the kernel level — no userspace proxy in the data path.

Per-Request Billing

Both engines bill per invocation. Metal adds per-GB-sec compute time. No hourly charges. No idle cost. You pay for what runs.

Written in Rust

The entire platform — API, daemon, proxy, CLI — is Rust. Postgres for state. NATS for events. Pingora for edge routing. No ORMs. No frameworks.

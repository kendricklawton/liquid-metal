Liquid Metal

A deploy platform for developers who ship compiled code. Push a binary, get an endpoint.

{# Problem #}

// problem

The mystery resource pool

Existing platforms force a choice: managed convenience with opaque resource sharing, or full control with management overhead. Your "2 vCPU" container might share a physical core with fifty other tenants. Performance is a lottery.

{# Approach #}

// approach

Dedicated isolation

Every Metal workload runs in its own Firecracker microVM with dedicated vCPU and RAM. No oversubscription. No noisy neighbors. What you allocate is what you get.

For lightweight workloads, Liquid (WebAssembly) provides sub-millisecond execution with per-invocation billing. Same binary, different runtime.

{# Stack #}

// zero-trust infrastructure

No Kubernetes

No managed orchestrators. Bare metal or death.

Firecracker + Jailer

VM-level isolation with seccomp and namespace sandboxing.

eBPF (Aya)

Per-tenant network isolation — TC egress classifiers block cross-tenant traffic at the kernel level.

cgroup v2

Triple-lock resource control: hypervisor limits, kernel I/O limits, and network shaping.

{# Built with #}

// built with

The entire platform is written in Rust. API, daemon, proxy, CLI — all Rust. Postgres for state. NATS JetStream for event delivery. Pingora for edge routing. No ORMs. No frameworks. Raw SQL and hand-rolled infrastructure.

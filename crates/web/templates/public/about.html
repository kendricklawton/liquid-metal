Liquid Metal

I built Liquid Metal because deploy platforms kept asking me for a Dockerfile. If you write Go, Rust, or Zig, you already have a binary. You shouldn't need to wrap it in a container just to run it. So I built a platform that takes your compiled output and runs it directly — on a dedicated VM or as serverless Wasm.

{# Problem #}

// problem

Deploy platforms weren't built for compiled languages

Lambda wants a runtime. Vercel wants JavaScript. Cloudflare Workers want V8. If you write Go, Rust, or Zig, you already have a binary — you shouldn't need to wrap it in a container or an SDK just to run it. Liquid Metal takes your compiled output and runs it directly, on real hardware or in a Wasm sandbox.

{# Approach #}

// approach

Two engines, one CLI

Metal runs your binary in a dedicated Firecracker microVM on bare metal hardware. Always on, no cold starts, fixed monthly price. Real Linux kernel, real networking, real isolation. For anything that compiles to x86_64-unknown-linux-musl.

Liquid runs your Wasm module in Wasmtime. Serverless — scale to zero, wake on request, sub-millisecond cold starts. Per-invocation billing. For APIs, webhooks, and data transforms that compile to wasm32-wasip1.

{# How it works #}

// how it works

Deploy

flux deploy builds your binary, uploads it to object storage, and provisions your service. No Dockerfiles. No YAML. One command.

Metal — always on

Metal VMs boot once, pass a health check, and stay running permanently. Traffic hits your binary directly — no wake-up, no cold start, no latency tax.

Liquid — wake on request

Liquid services scale to zero when idle. When a request arrives, Wasmtime cold-starts your module in under a millisecond. The user never waits.

{# Stack #}

// under the hood

Firecracker on Bare Metal

Metal services run in KVM-isolated Firecracker microVMs on dedicated bare metal servers. Always on, full Linux kernel, no containers, no neighbours sharing your CPU.

eBPF Tenant Isolation

Per-VM network isolation via TC egress classifiers. Cross-tenant traffic is blocked at the kernel level — no userspace proxy in the data path.

Billing

Metal is fixed monthly pricing — always on, no metering. Liquid bills per invocation with a generous free tier. No idle cost, no surprises.

Written in Rust

The entire platform — API, daemon, proxy, CLI — is Rust. I wrote all of it. Postgres for state. NATS for events. Pingora for edge routing. Firecracker for VMs. eBPF for isolation. No ORMs. No frameworks.

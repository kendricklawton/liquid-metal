Liquid or Metal?

Different tools for different jobs. Same CLI, same deploy flow.

Liquid

  • Serverless — scale to zero, wake on request
  • Wasm sandbox — no kernel, no filesystem
  • Sub-millisecond cold starts
  • Pay per invocation
  • Best for: APIs, webhooks, data transforms
  • Target: wasm32-wasip1

Metal

  • Dedicated VM — always on, fixed monthly price
  • Dedicated vCPU — no shared resources, no throttling
  • Real Linux — full kernel, filesystem, networking
  • No cold starts — always running
  • NVMe storage, KVM isolation, bare metal hardware
  • Best for: APIs, databases, background jobs, anything
  • Target: x86_64-unknown-linux-musl
